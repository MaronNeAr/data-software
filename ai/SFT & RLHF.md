## SFT 有监督微调
#### 第一部分 理论基石
###### 从 Next Token Prediction 到 Instruction Following
- **Base Model 的行为**：如果你问“如何写一个 Python 脚本？”，Base Model 可能会回你“第二个问题：如何写一个 Java 脚本？”，因为它认为你在给它提供一个列表
- **SFT 的干预**：SFT 的核心是将模型从“无序的续写者”转变为“有序的指令遵循者”。通过在 $(Prompt, Response)$ 这种高质量对齐数据上微调，模型学习到了特定的**条件概率分布**：
  - $P(Response \mid Prompt, \theta)$
- **关键技术点**：在代码实现层，虽然我们将 $Prompt + Response$ 拼接后整体输入模型，但在计算 **Cross Entropy Loss** （交叉熵损失）时，我们会将 $Prompt$ 部分对应的 Label 设为 `-100`（在 PyTorch 中默认忽略该值的 Loss 计算）。
- **核心结论**：SFT 训练时，模型只为它“回答”的那部分内容负责，而不为它“读到”的指令负责。

###### 激发（Elicitation）而非 学习（Learning）

- **知识来源**：绝大部分事实性知识（如“鲁迅是谁”）是在 Pre-training 阶段通过海量语料习得的。
- **SFT 的作用**：
  - **格式对齐**：学会用 JSON、代码块或 Markdown 回复。
  - **语气对齐**：学会专业、礼貌或幽默的口吻。
  - **能力解冻**：将预训练中压缩的万亿级参数，按照“指令-回答”的逻辑通路进行激活。

###### 参数高效微调 (PEFT) 的数学本质

- **核心公式**：假设原始预训练权重为 $W_0 \in \mathbb{R}^{d \times k}$，在微调时，我们不改变 $W_0$，而是学习一个增量 $\Delta W$。根据低秩分解理论：
  - $$\Delta W = B \cdot A$$，其中 $A \in \mathbb{R}^{r \times k}$，$B \in \mathbb{R}^{d \times r}$，且秩 $r \ll \min(d, k)$
- **为什么要这么做？**
  - **显存压缩**：如果 $r=8$，参数量通常能减少到原来的 **1/1000** 甚至更低。
  - **防止过拟合**：限制了模型修改权重的“自由度”，使其在小规模数据集上更稳定。
  - **可切换性**：Base 模型不动，只需要在线下加载不同的 Adapter（几十 MB），即可让同一个模型在“代码助手”和“文案专家”之间秒切。

###### 灾难性遗忘 (Catastrophic Forgetting)

- **现象**：当你为了让模型学会“写 SQL”而喂了大量的 SQL 专有数据后，模型可能会突然忘记如何写诗，或者通识推理能力下降。
- **成因**：由于微调数据分布过于单一，模型权重向特定区域剧烈偏移，破坏了预训练阶段形成的全局表征。
- **对策（理论指导实践）**：
  - **数据回放 (Data Replay)**：在 SFT 数据中混入 5%-10% 的预训练通用数据。
  - **正则化**：使用较小的学习率（通常在 $10^{-5}$ 或 $10^{-6}$ 级别）。

#### 第二阶段：数据工程（SFT 的“灵魂”）

###### 数据构造的“金字塔结构”

- **指令的多样性 (Diversity)**
  - **直接指令**：“请翻译这段话。”
  - **角色扮演**：“你现在是一名资深的后端工程师，请审查以下代码。”
  - **少样本提示 (Few-shot)**：给出 1-2 个例子，让模型模仿。
- **任务类型的覆盖**
  - **基础能力**：摘要、提取、翻译、问答。
  - **逻辑推理**：数学题、代码编写、逻辑悖论。
  - **安全性**：拒绝回答违法、暴力或偏见类问题。

###### 数据的获取与生成 (Data Sourcing)

- **人工标注**：最昂贵但最准确的方式。通常用于处理极具行业深度的任务（如医疗、法律或公司内部复杂的业务逻辑）。
- **Self-Instruct (模型自生成)**
  - **Seed Tasks**：人工写 175 个种子任务。
  - **Generate**：让强模型根据种子任务扩展出几万个新任务。
  - **Filter**：剔除重复和低质量的生成结果。
- **开源数据集的二次加工**：将社区开源的 Alpaca、ShareGPT、COIG 等数据集进行清洗、翻译和格式对齐。

###### 数据清洗与质量控制 (The Hard Work)

- **启发式过滤 (Heuristic Filtering)**

  - **长度过滤**：Response 太短（如“好的”、“不知道”）或太长（复读机）的数据直接丢弃。

  - **关键词黑名单**：剔除包含训练格式错误、低俗词汇或特定模型自我介绍（如“我是由 OpenAI 开发的”）的数据。

- **语义去重 (Semantic Deduplication)**：如果数据中包含 1000 条问“如何煮面”的指令，模型会产生偏见。利用 **Embedding 向量**计算余弦相似度，将意思高度接近的数据点进行聚类并去重，保证数据分布的**熵最大化**。

- **LLM-as-a-Judge (自动化打分)**：利用 GPT-4 或内部微调的高性能 Reward Model 对 SFT 数据对进行打分。

  - **维度**：**真诚度 (Honesty)**、**无害性 (Harmlessness)**、**有用性 (Helpfulness)**。

  - 只保留评分在 4.5/5.0 以上的数据进入训练集。

######  数据配比 (Data Mixing) 的艺术

- **通用能力保护**：加入 10%-20% 的通用对话数据（如维基百科摘要或日常闲聊），防止模型在学会写 SQL 的同时变傻。
- **任务平衡**：如果代码数据占 80%，问答占 20%，模型可能会在回答文本问题时也带上代码块。

#### 第三阶段：工程实战（核心技能）

###### 框架选型

- **入门级**：使用 **LLaMA-Factory**。它集成了数据处理、训练、合并权重和推理，是目前社区最友好的“一站式”工具，适合快速建立直观认识。
- **进阶级**：直接上手 **Hugging Face PEFT + Transformers + Accelerate**。这是工业界的标准库，能让你深入代码细节。
- **大厂级**：研究 **DeepSpeed**（ZeRO 优化）或 **Megatron-LM**，理解在大规模显卡集群上如何进行算力优化。

###### 关键实验

- **LoRA vs. QLoRA**：尝试在有限显存下（如单张 4090 或 A100）使用 4-bit 量化训练。
- **全量微调 (Full Fine-tuning)**：对比全量微调与 LoRA 在特定领域知识注入上的差异。
- **长文本微调**：尝试扩展模型的上下文长度（Context Length）。

#### 第四阶段：评估与性能分析（闭环）

- **客观评估**：使用 Benchmark（如 MMLU, CMMLU, C-Eval）跑分，观察模型基础能力是否下降（灾难性遗忘）。

- **主观评估**：构建自己的 Test Set，使用侧重特定业务场景的测试集进行人工评估或 LLM 对比。

- **训练监控**：学会看 Tensorboard 曲线。

  - **Loss 骤降**：可能是过拟合或数据过于简单。

  - **Loss 不降**：检查数据格式、学习率是否过大、Prompt 是否生效。

## RLHF 基于人类反馈的强化学习

#### 核心流程：经典的“三步走”战略

- **第一阶段**：SFT (Supervised Fine-Tuning)：在进行强化学习之前，模型必须先具备基本的说话能力。这正是你之前学习的 SFT 过程，产出一个 **SFT 模型**（也叫 Policy 策略）。

- **第二阶段**：RM (Reward Model，奖励模型训练)

  - **数据构造**：给模型一个 Prompt，让它生成多个结果（A, B, C, D），由人类对这些结果进行排序（如 $A > B > C$）。

  - **训练目标**：训练一个回归模型（通常是把 SFT 模型最后的线性层换成输出标量的层），使其预测的分数符合人类的排序逻辑。

  - **数学本质**：使用 **Bradley-Terry 模型** 将排序转化为概率，通过最大化胜出项与落败项之间的分差来训练。

- **第三阶段**：RL (Reinforcement Learning，强化学习对齐)

  - **环境反馈**：模型生成一个 Token 序列，RM 模型给出一个总分。

  - **约束机制**：为了防止模型为了刷高分而变得语无伦次（即“奖励作弊”），会引入一个 **KL 散度** 惩罚项，限制当前模型不要偏离原始 SFT 模型太远。

#### 关键组件与数学原理

- **奖励作弊 (Reward Hacking)**：模型发现 RM 模型的漏洞。例如，RM 可能觉得“长回复”更有礼貌，模型就会学会无论问什么都回复长篇大论的废话来刷分。理解这一点能帮你理解为什么 KL 惩罚至关重要。

- **PPO 算法**

  - **Actor-Critic 架构**：谁在做动作（Actor），谁在评估动作的好坏（Critic）。

  - **重要性采样 (Importance Sampling)**：如何利用旧策略的数据更新新策略。

- **评估闭环**：学习如何使用 Elo Rating（类似等级分系统）或者 GPT-4-as-a-Judge 来评估对齐后的模型是否真的比 SFT 模型更符合人类价值观。

#### 工程实现与工具链

- **TRL (Transformer Reinforcement Learning)**：Hugging Face 官方库，目前学习 PPO 最标准的起点。
- **DeepSpeed-Chat**：微软推出的三阶段全流程框架，重点学习它如何管理四个模型（Actor, Reference, Critic, Reward）在显存中的分布。
- **Ray / vLLM**：在大规模强化学习中，推理速度是瓶颈。学习如何利用 vLLM 加速 Actor 的生成过程。

#### 绕不过去的新高峰：DPO (Direct Preference Optimization)

- **为什么学 DPO？** PPO 太不稳定，且需要同时维护 4 个模型，显存开销极大。
- **原理简述**：DPO 通过数学变换，直接在偏好数据上微调策略模型，**省去了训练奖励模型和复杂的 PPO 过程**。
- **学习重点**：理解 DPO 的 Loss Function 如何通过一个简单的概率比值来隐式地表示奖励。
